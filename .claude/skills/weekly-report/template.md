## 記載ルール

### 変更サマリーのカウント方法
- **新規**: 今週新たに発生した項目・プロジェクト
- **更新**: 先週から状況が変化した項目
- **完了**: 今週完了した項目・プロジェクト

### 出典タグの付け方
各進捗には出典を明記する：
- `\[src: MeetingNote YYYY-MM-DD\]` - ミーティングノートからの情報
- `\[src: D社ページ YYYY-MM-DD\]` - プロジェクトページからの情報
- `\[src: AI Coding DB#<task-id>\]` - タスクDBからの情報

### 確証が薄い情報の扱い
情報ソースが曖昧、未確定な予定、相互矛盾がある場合は末尾に `<確認してください>` を付与：
```
推論APIのスループット計測 500rps→700rps（暫定値）\[src: MeetingNote 2025-09-18\] <確認してください>
```

### MBO項目の更新ルール
- 進捗なしの場合: 「特に進捗なし」または「継続」と記載
- 新規タスク: 追記
- 状況変化: 本文を直接更新（上書き）
- 完了タスク: 「完了」と記載または削除
