# Weekly Report Review

週報に対して松尾先生の視点からフィードバックを生成し、Notionコメントとして記録する。

## Workflow

```
1. 週報の取得 → 2. フィードバック生成 → 3. Notionコメント投稿
```

### Step 1: 週報の取得

引数で指定されたページID、または直近の週報を検索して取得。

```
Notion検索 → 週報ページの本文を取得
```

### Step 2: フィードバック生成

松尾先生の視点で以下の観点からコメントを生成:

#### レビュー観点

| 観点 | チェックポイント |
|------|-----------------|
| **出口の明確さ** | 各活動の「出口」（具体的な成果物・ゴール）が明確か。抽象的な表現で終わっていないか |
| **進捗の具体性** | 「検討中」「継続」だけでなく、具体的な進捗や数値があるか |
| **優先順位** | 重要な案件が適切に優先されているか、リソース配分は適切か |
| **リスク認識** | 潜在的な問題点やリスクが見落とされていないか |
| **次のアクション** | 翌週に何をするかが明確か |

#### コメントフォーマット

```
【松尾先生視点レビュー】

■ 良い点
- ...

■ 要確認・改善点
- [項目名]: 具体的な指摘内容
  → 推奨アクション

■ 質問事項
- ...
```

### Step 3: Notionコメント投稿

`mcp__notion__notion-create-comment` を使用してコメントを投稿:

```json
{
  "parent": {"type": "page_id", "page_id": "<週報ページID>"},
  "rich_text": [{"type": "text", "text": {"content": "<コメント内容>"}}]
}
```

## 松尾先生のレビュー傾向

過去のフィードバックから抽出した傾向:

1. **「出口」を重視**: 活動の最終的なアウトプット・成果物を明確にすることを求める
2. **調査のための調査を警戒**: 「やってよかったですね」で終わらない、実務に活用される形を求める
3. **具体的な提言を期待**: 抽象的な方針ではなく、実行可能な具体策を求める
4. **数値・期限の明確化**: 定量的な目標設定や期限を重視

## Arguments

| Parameter | Type | Description |
|-----------|------|-------------|
| `page_id` | string | 週報のNotionページID。省略時は直近の週報を検索 |

## Example Output

```
【松尾先生視点レビュー】

■ 良い点
- 体制強化でNDA5件締結と具体的な数値が示されている
- AIツールベンチマークの改善方向が明確

■ 要確認・改善点
- [業務可視化]: 「自動連携システムの構築へ」とあるが、具体的にいつまでに何ができる状態を目指すのか不明確
  → 成果物と期限を明確にする

- [AIツールベンチマーク]: 「推奨ツール早見表の作成」は良いが、これがどのように提案・デリバリに活用されるかの導線が見えない
  → 実際の提案プロセスへの組み込み方を明確にする

■ 質問事項
- さとふるの審議が通った場合の体制は準備できているか？
```
