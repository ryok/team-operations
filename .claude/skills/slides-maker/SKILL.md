# スライド作成スキル（slides-maker）

松尾研究所のPowerPointテンプレートに準拠した、プロフェッショナルな日本語プレゼンテーションを作成する。

## トリガー条件

- ユーザーが日本語の生データ、メモ、リサーチ、ドキュメントを提供し、プロフェッショナルなスライドを求めた場合
- 経営ブリーフィング、取締役会資料、投資家向けプレゼンテーション
- チャートやフレームワーク、テーブルを含む高密度の分析スライドの要求

## 前提条件

作業開始前に、必ず以下のスキルファイルを読み込むこと：
```
Read slides-maker/references/pptx/SKILL.md
Read slides-maker/references/pptx/editing.md
```

次に、松尾研究所デザインシステムのリファレンスを読み込む：
```
Read slides-maker/references/design-system.md
```

## テンプレートファイル

テンプレート: `references/松尾研究所テンプレ_v2_DLしてお使いください.pptx`

このテンプレートのスライドマスターが自動提供するもの：
- ロゴ（右上）
- コピーライト（下部中央）「©︎MATSUO INSTITUTE, INC.」
- セパレーターライン
- テーマカラー

**重要**: テンプレートを使用することで、ロゴ・コピーライト・セパレーターラインを手動追加する必要がなくなる。

## レイアウト選択ガイド

テンプレートには6つのレイアウトが用意されている。スライドの種類に応じて適切なレイアウトを選択すること：

| Layout# | 名前 | 用途 | 主なプレースホルダー |
|---------|------|------|---------------------|
| 0 | Title Slide | 表紙 | idx=0:タイトル, idx=1:日付, idx=10:部署, idx=11:宛名, idx=12:氏名 |
| 1 | Title and Content | コンテンツ（本文PH付き） | idx=0:タイトル, idx=1:本文, idx=12:ページ番号 |
| 2 | 1_Title and Content | コンテンツ（自由配置） | idx=0:タイトル, idx=12:ページ番号 ※本文PHなし |
| 3 | Custom Layout | セクション区切り | idx=0:タイトル（中央配置） |
| 4 | 2_Custom Layout | ブランク＋ページ番号 | idx=11:ページ番号 |
| 5 | 1_Custom Layout | ブランク＋ロゴ | なし |

### レイアウト選択の指針

- **表紙**: Layout 0（Title Slide）— プレースホルダーにタイトル・日付・部署・宛名・氏名を設定
- **コンテンツスライド（箇条書き中心）**: Layout 1（Title and Content）— 本文プレースホルダーに箇条書きを設定
- **コンテンツスライド（テーブル・チャート・自由配置）**: Layout 2（1_Title and Content）— タイトルのみPHで設定し、コンテンツは`add_table()`/`add_shape()`で自由配置
- **セクション区切り**: Layout 3（Custom Layout）— セクションタイトルのみ
- **ブランク（ページ番号あり）**: Layout 4
- **ブランク（ロゴのみ）**: Layout 5

## ワークフロー

### 1. ユーザー資料の分析

アップロードされたファイルまたは提供されたテキストを調査し、以下を特定する：
- **キーメッセージ**: 主要な2〜3つの結論・洞察は何か？
- **データポイント**: 数値、パーセンテージ、トレンド、比較
- **適用可能なフレームワーク**: SWOT、2x2マトリクス、ウォーターフォール、ファネル、タイムライン
- **階層構造**: 全体的なナラティブアーク（物語構造）

### 2. デッキ構成の計画

松尾研究所テンプレートに従い、以下の構成で計画する：

1. **タイトルスライド** — Layout 0 を使用。テーマ、クライアント名（「○○ 御中」）、日付、部署名、作成者名。
2. **Agenda** — Layout 2 を使用。ナンバーバッジ付き目次。tx2カラー（44546A）の四角バッジ。
3. **セクション区切り** — Layout 3 を使用。章番号＋タイトルのみ。
4. **コンテンツスライド** — Layout 1 または Layout 2 を使用。分析、データ、フレームワーク。以下を組み合わせる：
   - データビジュアライゼーションスライド（チャート＋注釈）→ Layout 2
   - フレームワークスライド（2x2マトリクス、比較テーブル）→ Layout 2
   - 箇条書き中心のスライド → Layout 1
   - 深堀りスライド（マルチカラム＋補足データ）→ Layout 2
5. **示唆 / 提言** — 構造化されたテイクアウェイ、優先度付きアクション。
6. **付録**（任意）— 補足データテーブル、方法論ノート。

各コンテンツスライドには必ず**アクションタイトル**を付けること。トピックラベルではなく、洞察を述べる文。

**アクションタイトルの例：**
- ✅ 「デジタル売上は前年比34%増、主にエンタープライズ契約が牽引」
- ❌ 「売上概要」

### 3. python-pptxによるテンプレートベースのスライド構築

テンプレートを開き、既存サンプルスライドを削除してから新しいスライドを追加する。

```python
from pptx import Presentation
from pptx.oxml.ns import qn

# テンプレートを開く
prs = Presentation('references/松尾研究所テンプレ_v2_DLしてお使いください.pptx')

# 既存スライド（サンプル）を全削除
while len(prs.slides._sldIdLst) > 0:
    rId = prs.slides._sldIdLst[0].get(qn('r:id'))
    prs.part.drop_rel(rId)
    prs.slides._sldIdLst.remove(prs.slides._sldIdLst[0])

# レイアウトからスライドを追加
slide = prs.slides.add_slide(prs.slide_layouts[2])  # Layout 2: 自由配置コンテンツ
slide.placeholders[0].text = "スライドタイトル"
```

`references/design-system.md` のデザインシステムに厳密に従うこと。主要原則は以下の通り：

**カラーパレット：**
- 背景: `FFFFFF`（ホワイト）— テンプレートのスライドマスターが提供
- プライマリテキスト: `000000`（ブラック）
- チャートプライマリ: `4472C4`（accent5ブルー）
- テーマカラー: `5B9BD5`（accent1）, `ED7D31`（accent2）, `FFC000`（accent4）, `70AD47`（accent6）
- テーブルヘッダー: `334F93`（accent5 lumMod 75%）

**タイポグラフィ：**
- スライドタイトル: Hiragino Kaku Gothic Pro W6, 26pt, ボールド
- 本文: Hiragino Kaku Gothic Pro W3, 12-24pt
- データ数値: Calibri, 10-12pt, ボールド
- ソースライン: Hiragino Kaku Gothic Pro W3, 10.5pt, 「出典）...」形式

**テンプレートが自動提供するもの（手動追加不要）：**
- ロゴ（右上）
- コピーライト（下部中央）「©︎MATSUO INSTITUTE, INC.」
- セパレーターライン
- テーマカラー定義

### 4. チャート＆ビジュアライゼーション ガイドライン

python-pptxを使用して、テンプレート準拠のスタイリングでチャートを生成する：

- **棒グラフ/カラムチャート**: accent5ブルーをプライマリ、テーマカラーをセカンダリに。3D不可。クリーンな値ラベル。
- **積み上げ棒グラフ**: テーマカラーパレットをセグメントに使用。
- **折れ線グラフ**: 細いライン（2pt）、データポイントマーカー、クリーンなグリッド。
- **テーブル**: ヘアラインボーダー（`D0D0D0`, 0.5pt）。ヘッダー行はaccent5Dark（`334F93`）白テキスト。本文12pt。行の背景色を交互に。
- **ウォーターフォールチャート**: 積み上げ棒グラフで構築。

### 5. フレームワークスライド

2x2マトリクス、比較フレームワーク、戦略ダイアグラムの場合：
- シェイプ（矩形＋ライン）で構築し、画像は使わない
- 細いライン（0.5-1pt）を`D0D0D0`で使用
- 軸ラベルはマトリクスの外側に明確に配置
- 各象限にテーマカラーのティント背景
- 象限ラベルはW6ボールド、説明文はW3レギュラー
- 日本語テキストは必ずテキストボックスの幅を十分に確保（折り返し対策）
- コールアウトは矩形を使用（楕円不可）
- ナンバーバッジは四角形、tx2カラー（44546A）

### 6. 日本語特有の注意点

- **フォントの指定**: 見出しにはHiragino Kaku Gothic Pro W6、本文にはHiragino Kaku Gothic Pro W3を使用。英数字やデータ数値にはCalibrを使用。
- **フォントフォールバック**: Windows環境ではMeiryo UIまたはSegoe UIにフォールバック。
- **文字サイズ**: テンプレートの階層に従う（L1:24pt → L2:20pt → L3:16pt → L4:12pt）。
- **テキストボックスの幅**: 日本語テキストは英語より幅が必要。余裕を持った幅設定を行う。
- **句読点**: 全角の「。」「、」を使用する。
- **敬語レベル**: ビジネス文書として適切な敬語レベルを維持する。「です・ます」調を基本とする。
- **数字表記**: 大きな数字は漢数字（億、万）と算用数字を適切に組み合わせる（例：1.2兆円、340億円）。

### 7. QAプロセス

標準のPPTX QAプロセスに従う：
1. PDFに変換 → 画像化
2. 全スライドを目視で検査：
   - テキストのオーバーフローや重なり
   - チャートの可読性
   - マージンとアライメントの一貫性
   - カラーの一貫性（テンプレートのテーマカラー準拠）
   - 全コンテンツスライドにアクションタイトルがあるか
   - スライドマスターの要素（ロゴ・コピーライト・セパレーターライン）が正しく表示されているか
   - 日本語テキストの文字化けや切れがないか
   - フォントのフォールバックが発生していないか
3. 問題を修正し、再検証

## デザイン・アンチパターン（避けるべきこと）

- トピックラベルのみの汎用タイトル（アクションタイトルでないもの）
- ビジュアル要素のない箇条書きだらけのスライド
- 3つ以上のフォントファミリーの混在（Hiragino W6/W3 + Calibriの3種まで）
- ソース引用のないチャート
- ホワイトスペースのない詰め込みレイアウト
- テーマカラーパレット外の色の使用
- 情報を伝えない装飾要素
- ロゴ・コピーライト・セパレーターラインの手動追加（テンプレートが自動提供する）
- 楕円形のコールアウト（矩形を使用すること）
- 円形のナンバーバッジ（四角形を使用すること）
- Yu Mincho / Yu Gothicフォントの使用（Hiragino Kaku Gothic Proが正しい）
